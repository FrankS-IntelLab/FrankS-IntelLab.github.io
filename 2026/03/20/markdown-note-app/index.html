Building a Markdown Note-Taking App

After trying every note app on the market, I built my own. Here’s why and how.

The Problem

Most note apps are either too heavy, require internet, or lock your data in proprietary formats. I wanted something that stores plain .md files I can read anywhere.

What I Built

A local-first Markdown editor with:

  • File-system based storage
  • Instant full-text search
  • Zero cloud dependency

Key Decisions

Plain files over databases. Your notes are just files in a folder. No export needed — they’re already portable.

No sync built-in. Use whatever sync tool you already trust: iCloud, Dropbox, or git.

Takeaway

The best tool is the one that gets out of your way. Sometimes that means building it yourself.
